The Marimba v1.1.0 release delivers significant improvements to EXIF metadata handling, memory efficiency for large datasets, and iFDO output size, alongside a comprehensive test suite and a range of quality-of-life improvements for packaging workflows.

## What's New in v1.1.0

- **PyExifTool Integration**: Replaced `piexif` with `PyExifTool` for all EXIF metadata writing. This resolves long-standing issues with non-standard EXIF structures silently failing or corrupting metadata, and adds support for embedded thumbnails and a much wider range of image formats.
- **Memory-Aware Batch Processing**: Image processing now samples actual files to estimate per-image memory requirements and calculates a safe chunk size from available system RAM. Adaptive retry logic halves the batch size on memory pressure, preventing out-of-memory crashes on large high-resolution datasets.
- **iFDO Auto-Deduplication**: Fields that are identical across all images in a collection (sensor, platform, deployment event, license, etc.) are automatically promoted to the `image-set-header`, removing duplication from individual item entries and reducing output file size.
- **Packaging Safety Improvements**: Marimba now detects read-only files before packaging begins and fails immediately with a clear error, rather than partway through a long run. Hard-linked files trigger a warning prompt before proceeding, since EXIF writes propagate to all linked paths.
- **Non-Interactive Packaging**: A new `--accept-defaults` flag enables fully automated packaging runs without interactive prompts, suitable for CI/CD pipelines and scripted workflows.
- **System Dependency Framework**: A structured dependency checker provides platform-specific installation instructions for required system tools (ExifTool, FFmpeg) when they are missing, replacing generic error messages.
- **Marimba Version in Dataset Summaries**: Dataset summary files now include the Marimba version used to produce them, improving provenance tracking.
- **iFDO Datetime Compliance**: `image-datetime` values are now serialized using the iFDO spec-defined format (`%Y-%m-%d %H:%M:%S.%f`) rather than ISO 8601, ensuring output files are fully compliant with the iFDO v2.1.0 schema. Existing files with ISO 8601 datetime strings continue to load without error.
- **Comprehensive Test Suite**: Coverage grows from minimal to nearly 1,000 tests at 80%+, with unit, integration, and end-to-end tests across every major module.
- **Performance Improvements**: A repository-wide performance pass roughly halves cold CLI startup time via lazy imports of heavy native dependencies, accelerates EXIF metadata writing through a batched ExifTool session shared across each pipeline run, parallelises per-file dataset summarisation and S3 distribution uploads, consolidates redundant dataset-tree walks into a single pass, and caches pipeline module imports within a process.
- **Codebase Hardening**: A codebase-wide review pass introduces a `MarimbaError` exception base with narrowed CLI catch-alls, ensures CLI failures surface with non-zero exit codes, tightens the pipeline install lifecycle, aggregates per-pipeline update failures rather than swallowing them, and improves concurrency safety in the pipeline loader.
- **CI and Regression Infrastructure**: Marimba now ships with GitHub Actions CI and nightly workflows, Dependabot coverage for pip and GitHub Actions, a tracked `uv.lock` for reproducible dev and CI environments, and a tiered end-to-end regression harness against the `mritc-demo` dataset that runs as both a pre-push gate and on every CI run.

## Breaking Changes

- **Python 3.12+ required** (previously 3.10+)
- **`exiftool` system binary required** for EXIF metadata writing. See installation instructions below.
- **`piexif` dependency removed** and replaced by `PyExifTool`
- **`PipExecutor` removed** and replaced by `UvExecutor`; pipelines must be managed through `uv`
- **`image-datetime` format change**: iFDO files produced by this version will use the space-separated datetime format (`2019-03-04 08:37:24.000000`) instead of ISO 8601 (`2019-03-04T08:37:24Z`). Files produced by earlier versions continue to load correctly.
